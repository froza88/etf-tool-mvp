## 一、现有数据源深度分析

### 1.1 非凸科技 (market.ft.tech)

**接入方式**: Skill调用 (`ftshare-market-data`)

| 维度 | 评分 | 说明 |
|------|------|------|
| **数据覆盖** | ⭐⭐⭐⭐ | ETF详情、K线、成分股、规模、收益率 |
| **数据质量** | ⭐⭐⭐⭐ | 收益率/规模准确，但volume仅6.1% |
| **稳定性** | ⭐⭐⭐ | 偶发超时，需重试机制 |
| **更新频率** | ⭐⭐⭐⭐ | 日频，T+1 |
| **成本** | ⭐⭐⭐⭐⭐ | 免费 |
| **合规性** | ⭐⭐⭐⭐ | 需确认授权范围 |

**优点**:
- ✅ ETF专业数据全面（含1Y/3Y/6M收益率）
- ✅ 提供OHLC K线数据（可自算回撤/夏普/波动率）
- ✅ 成分股数据完整（含股票代码+名称）
- ✅ 免费使用

**缺点**:
- ❌ `volume`字段覆盖率极低（6.1%），流动性指标失真
- ❌ `change_rate_3y`覆盖率不稳定（部分ETF缺失）
- ❌ API偶发超时（需3-5次重试）
- ❌ 无历史持仓数据（只有当前成分股）

**适用场景**: 主数据源，ETF基础信息+K线+成分股

### 1.2 AKShare (东方财富)

**接入方式**: Python库 (`pip install akshare`)

| 维度 | 评分 | 说明 |
|------|------|------|
| **数据覆盖** | ⭐⭐⭐⭐⭐ | A股/ETF/基金/宏观全量数据 |
| **数据质量** | ⭐⭐⭐⭐ | 字段位置偶变（iloc索引风险） |
| **稳定性** | ⭐⭐⭐⭐ | 本地库，无API限流，但依赖东方财富网页结构 |
| **更新频率** | ⭐⭐⭐⭐⭐ | 实时/日频，取决于函数 |
| **成本** | ⭐⭐⭐⭐⭐ | 完全免费开源 |
| **合规性** | ⭐⭐⭐ | 爬虫性质，存在结构性风险 |

**优点**:
- ✅ 数据维度最全（行情/财务/宏观/期货/期权）
- ✅ 提供ETF持仓权重（`fund_portfolio_hold_em`）
- ✅ 提供历史日线数据（`fund_etf_hist_em`）
- ✅ 无API限流，本地直接调用
- ✅ 免费开源，社区活跃

**缺点**:
- ❌ 字段位置不稳定（用`iloc`硬编码，东方财富改版即失效）
- ❌ 无官方SLA，数据延迟不可控
- ❌ 部分函数需要模拟浏览器（依赖Selenium/Playwright）
- ❌ 爬虫性质，存在被封IP或法律风险

**适用场景**: 辅助数据源，持仓权重+历史日线+全市场列表

### 1.3 盈米基金 (Yingmi)

**接入方式**: Skill CLI (`yingmi-skill-cli`)

| 维度 | 评分 | 说明 |
|------|------|------|
| **数据覆盖** | ⭐⭐⭐ | 风险指标（夏普/回撤/波动率） |
| **数据质量** | ⭐⭐⭐⭐ | 专业计算，但字段映射不明确 |
| **稳定性** | ⭐⭐ | 每日额度限制（429限流） |
| **更新频率** | ⭐⭐⭐ | 日频，但额度用完即停 |
| **成本** | ⭐⭐⭐⭐⭐ | 免费（有额度限制） |
| **合规性** | ⭐⭐⭐⭐ | 需注册账号 |

**优点**:
- ✅ 专业风险指标（夏普比率、最大回撤、年化波动率）
- ✅ 支持多周期（1Y/2Y/3Y）
- ✅ 计算逻辑专业（考虑了分红再投资）

**缺点**:
- ❌ **每日额度限制**（429 Too Many Requests），大批量数据无法一次跑完
- ❌ 字段名映射不明确（"投资性价比"≠夏普比率？）
- ❌ 无本地缓存，重复请求浪费额度
- ❌ 仅支持风险指标，无价格/持仓数据

**适用场景**: 补充数据源，专业风险指标（夏普/回撤/波动率）

## 三、Skill市场数据源推荐

基于已安装的Skills，推荐以下数据源用于ETF工具增强：

### 3.1 强烈推荐

#### ① WeStock Data (westock-data)
**类型**: Plugin  
**覆盖**: A股/港股/美股个股/指数/ETF  
**能力**: 实时行情、K线/分时、财务报表、三表分析  
**适用**: 替换AKShare的`fund_etf_hist_em`，获取更稳定的ETF历史数据  
**优势**: 字段稳定、API化、不依赖网页结构  
**接入**: `Skill: westock-data`

#### ② iFinD 金融数据 (ifind-mcp / tonghuashun-ifind)
**类型**: Skill  
**覆盖**: A股/基金/宏观/新闻  
**能力**: 实时行情、财务数据、F10资料  
**适用**: 需要稳定字段的ETF基础数据  
**优势**: 同花顺官方数据源，字段极稳定  
**接入**: `Skill: ifind-mcp`

#### ③ MX Finance Data (mx-finance-data / mx-findata)
**类型**: Skill  
**覆盖**: A股/ETF/债券/港美股/基金  
**能力**: 自然语言查询金融数据  
**适用**: 快速查询单只ETF详情，无需写代码  
**优势**: 支持自然语言查询，降低使用门槛  
**接入**: `Skill: mx-finance-data`

### 3.2 有条件推荐

#### ④ 国泰海通金融数据 (lingxi-financialsearch-skill)
**类型**: Skill  
**覆盖**: A股实时行情、公司信息、F10  
**能力**: 自然语言查询  
**适用**: 补充ETF发行公司信息  
**注意**: 需要国泰海通账户权限

#### ⑤ Investment Data (investment-data)
**类型**: Skill  
**覆盖**: A股投资数据  
**能力**: 日终数据、高质量投资指标  
**适用**: 获取计算好的投资指标（如ROE、PE等）  
**注意**: 数据质量高但覆盖范围需确认

## 四、数据源架构优化建议

### 当前架构

```
ETF数据层
├── 非凸科技 (主) → 价格/收益率/规模/成分股
├── AKShare (辅) → 持仓权重/历史日线
└── 盈米 (补) → 风险指标（夏普/回撤/波动率）
```

### 优化后架构

```
ETF数据层 v2
├── 非凸科技 (主) → 价格/收益率/规模/成分股
├── AKShare (辅) → 持仓权重/全市场列表
├── 盈米 (补) → 风险指标（带本地缓存）
├── WeStock Data (增强) → 稳定的历史K线/分时
└── iFinD (备用) → 字段稳定的官方数据
```

### 具体优化方案

| 优先级 | 优化项 | 预期收益 |
|--------|--------|---------|
| P0 | **用WeStock Data替换AKShare的历史K线** | 字段稳定，不再依赖iloc |
| P1 | **用iFinD补充ETF基础信息** | 字段稳定性大幅提升 |
| P1 | **盈米增加本地缓存** | 减少429，支持增量更新 |
| P2 | **AKShare仅用于持仓权重** | 降低字段变动风险 |
| P2 | **增加数据质量校验层** | 自动检测异常值/缺失值 |

## 六、结论

1. **当前架构基本可用**，但存在3个核心痛点：
   - 盈米429限流 → 已优化（本地缓存+增量更新）
   - AKShare字段不稳定 → 建议引入WeStock Data/iFinD
   - volume覆盖率低 → 建议从AKShare补充

2. **短期（本周）**：完成盈米缓存优化，监控AKShare字段变动
3. **中期（本月）**：引入WeStock Data作为历史K线主源
4. **长期（下月）**：评估iFinD接入，构建多源校验机制
