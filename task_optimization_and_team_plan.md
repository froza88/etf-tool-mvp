## 二、优化分析：合并与去重

### 优化原则
1. **已完成任务** → 从待执行清单中移除（8项）
2. **已放弃任务** → 标记关闭（1项：盈米字段映射）
3. **同模块合并** → 相关任务打包为一个工作单元
4. **依赖排序** → 前置任务先做，后续任务后做
5. **长期任务拆阶段** → 把远期目标拆为本周可做的第一步

### 合并清单

| 合并前 | 合并后 | 原因 |
|--------|--------|------|
| #4 修复enrich_missing_fields.py + #5 修复data_absorber.py | **代码清理：修复/删除半成品脚本** | 同属代码债务清理 |
| #12 部署PA + #13 测试pre_deploy.sh + #15 检查PA数据同步 | **PA部署与验证** | 部署流程一体化 |
| #14 运行daily_update + #16 扩展历史缓存 + #18 迁移历史K线 | **数据全量更新与迁移** | 同一条数据流水线 |
| #23 分布式缓存架构 + #24 集成数据更新脚本 | **分布式缓存架构实施** | 架构设计与实现一体 |
| #25 PA定时任务 + #34 合并自动化任务 + #35 定时+告警 | **自动化运维体系** | 运维任务打包 |
| #27 加载速度优化 + #28 SQLite替代JSON | **数据存储升级** | 同属存储层优化 |
| #42 通用框架重构 + #38 DataAbsorber + #41 增量更新 | **通用数据引擎重构** | 架构升级一条线 |
| #43-46 扩展股票/基金/期货/外汇 | **泛金融工具扩展** | 长期规划打包 |
