# etf-compare-skill 卡比兽比卡 ETF 实时对比

## 业务流程图

```
用户意图
  │
  ├─ 模糊查询（"医药ETF"）────────────→ searchETF → ETF搜索结果卡片
  │
  ├─ 明确代码（"512010"）──────────────→ getETFDetail → ETF详情卡片
  │
  ├─ 对比场景（"医药和医疗哪个好"）─────→ compareETF → 多维对比表格卡片
  │
  ├─ 排名查询（"今年收益最高的ETF"）────→ rankETF → 排名列表卡片
  │
  └─ 快捷分组（"黄金ETF对比"）──────────→ quickCompare → 预设组合对比卡片
```

> **Agent 对比场景必调 compareETF**，禁止仅凭 searchETF 结果口头对比。
> **searchETF 最多返回10条**，如需更多结果引导用户缩小范围。
> **排名类查询必须先确定排序字段**（收益、规模、费率），不确定时反问用户。

## 原子接口依赖关系

| 接口 | 作用 | 前置条件 |
|------|------|----------|
| searchETF | 按名称/代码/分类搜索 ETF | — |
| compareETF | 对比 2-5 只 ETF 全维度指标 | 已有 2+ 个有效代码 |
| rankETF | 按指定字段排名 TOP N | 用户明确了排序维度 |
| getETFDetail | 查看单只 ETF 完整信息 | 已有有效代码 |
| quickCompare | 加载预设对比组 | 用户提到的场景有预设组 |

## 业务约束（跨接口铁律）

### 1. 数据真实性
- 所有数据来源于上交所/深交所官方公示，经持牌数据商交叉验证。
- 1685 只全市场 ETF，覆盖股票/债券/商品/货币/跨境全品类。

### 2. 输出规范
- 对比结果优先以 **结构化表格** 展示（规模/费率/收益/风险/持仓）。
- 数字统一格式：规模用"亿"、费率和百分比保留小数点后两位。
- 禁止对 ETF 做买入/卖出建议，仅提供客观数据。

### 3. 免责声明
- 每次涉及具体数据的回复末尾，必须包含：「数据仅供参考，不构成投资建议。投资有风险，决策需谨慎。」
