# etf-compare-skill 卡比兽比卡 ETF 实时对比

## 业务流程

```
用户意图
  │
  ├─ 模糊意图（"最近医药ETF怎么样"）────→ searchETF ──→ 搜索结果文本
  │
  ├─ 明确代码（"512010"）────────────────→ getETFDetail ──→ 单只详情文本
  │
  ├─ 对比意图（"医药和医疗哪个好"）───────→ compareETF ──→ 多维对比表格文本
  │
  ├─ 排名意图（"今年收益最高的ETF"）───────→ rankETF ──→ 排名列表文本
  │
  └─ 快捷场景（"黄金ETF对比"）───────────→ quickCompare ──→ 复用 compareETF
```

> **Agent 对比场景必须调用 compareETF**，严禁仅凭 searchETF 返回的片段数据口头对比。
> **排名类查询若用户未指定排序维度（"最好的"），需先反问用户**：想按收益、规模还是费率排名？
> **所有数据输出末尾必须附带免责声明**。

## 接口依赖关系

| 接口 | 作用 | 前置条件 |
|------|------|----------|
| searchETF | 按关键词搜索 ETF 列表 | — |
| compareETF | 对比 2-5 只 ETF 全维度指标 | searchETF 或用户提供的 2+ 个有效代码 |
| rankETF | 按字段排名 TOP N | 用户明确指定了排序维度 |
| getETFDetail | 单只 ETF 完整档案 | 已有有效代码（来自 searchETF 或用户提供） |
| quickCompare | 加载 12 组预设对比 | 用户提到的场景匹配预设组名称 |

## 跨接口铁律（全 Skill 生效）

### 1. 输出形态
- 所有成功返回必须向用户展示数据，不得仅回复"已查询"而无内容。
- 对比结果以 Markdown 表格呈现；搜索/排名结果以编号列表呈现。
- 禁止对 ETF 做买入/卖出/推荐建议，仅提供客观数据。

### 2. 数据来源
- ETF 代码必须来自 searchETF 返回的 `code` 字段原值，或用户直接输入的有效代码。
- 禁止从用户自然语言（如"那个医药的"）推断代码，应调用 searchETF 获取。

### 3. 免责声明
- 每次输出具体数据后，必须包含：「⚠️ 数据仅供参考，不构成投资建议。投资有风险，决策需谨慎。」

### 4. 执行顺序
- compareETF 调用前必须确认至少有 2 个有效代码。
- quickCompare 失败时不应重试，应引导用户走 searchETF → compareETF 流程。
