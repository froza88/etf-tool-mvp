### 1.2 数据缺失根因分析

1. **year_3_return = 0%**
   - `build_standard_data.py` 中自算逻辑仅针对 `year_1_return == 0` 的Top500 ETF
   - 非凸API已提供 year_1_return 的ETF被跳过自算，导致3年收益未补充
   - **修复建议**: 将自算逻辑改为"year_3_return == 0"作为触发条件，或对所有无3年数据的ETF批量计算

2. **volume = 6.1%**
   - 非凸API的 volume 字段覆盖率本身极低
   - 日均成交额是流动性的核心指标，缺失导致对比页"流动性"雷达图失真
   - **修复建议**: 用AKShare `stock_zh_index_daily` 或 `fund_etf_hist_em` 的成交额字段补充

3. **issuer_full = 0%**
   - `build_standard_data.py` 的标准化输出中**根本没有写入 `issuer_full` 字段**
   - 只写了 `issuer`（简称）和 `issuer_short`（也是简称）
   - **修复建议**: 在 build_standard_data.py 中增加 issuer_full 映射（可用 `issuer_extract` 模块的 `get_full_name`）

## 二、代码问题清单

### 2.1 数据与模板不一致

| 位置 | 问题 | 严重程度 |
|------|------|----------|
| `app.py` L19 | `fee_max` 筛选参数仍被API接收 | 中 |
| `etf_data.py` L83-85 | `fee` 字段筛选逻辑仍在，但数据已移除 | 中 |
| `templates/index.html` L453-454 | 前端仍有"最大费率"筛选输入框 | 中 |
| `templates/detail.html` L307 | 引用 `etf.issuer`（简称），但用户要求详情页顶部显示简写、发行人写全称 | 低 |
| `templates/compare.html` | 无 `fetchWithTimeout` 函数定义，但对比页调用 | **高** |

### 2.2 功能缺陷

1. **对比页 `fetchWithTimeout` 未定义**
   - `compare.html` 调用 `fetchWithTimeout('/api/risk/' + code, 10000)`，但该函数只在 `risk.html` 中定义
   - 用户直接从列表进入对比页时，获取盈米数据会报 `ReferenceError`
   - **修复**: 将 `fetchWithTimeout` 提取为全局函数或内联到 compare.html

2. **详情页走势图是模拟数据**
   - `detail.html` L381-395 使用 `generateData()` 随机生成走势，而非真实历史净值
   - 用户点击"1个月/3个月/1年/3年"看到的不是真实K线
   - **修复**: 接入真实净值API（非凸或AKShare）或移除走势图

3. **3年收益显示逻辑矛盾**
   - `index.html` L571-572: `y1 && !y3` 显示"不足3年"，但 year_3_return=0 可能是因为数据缺失，而非真的不足3年
   - 部分ETF成立超过3年但 year_3_return=0（数据缺失被误判为"不足3年"）
   - **修复**: 用 issue_date 判断是否满3年，再结合 year_3_return 是否存在

4. **风险指标页的自算数据不完整**
   - `risk.html` L127-133: `threeYear` 的自算数据中波动率/回撤/夏普/卡玛全部为0
   - 自算指标文件 `etf_calculated_metrics.json` 只计算了1年指标

### 2.3 代码冗余

- `templates/screening-demo-v2.html` 是演示页面，步骤2的"规模>=5亿"与步骤1的"10-150亿"重叠，逻辑冗余
- `v` 文件（空文件）在git未跟踪列表中，应删除

## 三、用户体验评估

### 3.1 可用性亮点

- 首页筛选器支持关键词、类型、分类、规模、收益率多维筛选
- 对比页雷达图+柱状图可视化直观，支持指标切换
- 风险指标页支持1/2/3/5年周期切换，数据源可切换（自算/盈米）
- 移动端有响应式适配（隐藏部分列、底部对比栏适配）
- 数据出处声明清晰（页脚标注数据来源和免责）

### 3.2 体验痛点

| 痛点 | 影响 | 建议 |
|------|------|------|
| 列表页"近3年"列大面积显示"不足3年"或空白 | 用户无法横向对比长期收益 | 优先修复3年收益数据 |
| 详情页走势图是随机生成的 | 误导用户以为是真实走势 | 接入真实净值或移除 |
| 日均成交额90%缺失 | 流动性指标形同虚设 | 补充数据源或隐藏该字段 |
| 费率筛选框仍存在但无效 | 用户筛选后无结果，困惑 | 移除前端费率筛选 |
| 对比页盈米数据获取失败无提示 | 用户以为数据加载完成 | 增加失败Toast提示 |
| 无基金管理人全称 | 用户无法识别机构背景 | 补充 issuer_full |

## 四、部署与运维状态

- **部署平台**: PythonAnywhere (froza.pythonanywhere.com)
- **Git状态**: 干净（仅一个空文件 `v` 未跟踪）
- **数据更新**: 全手动流程（运行脚本 → 生成JSON → git push → 触发WSGI重启）
- **自动化**: 无定时任务，无数据新鲜度监控
- **日志**: 无应用日志文件，仅脚本执行日志

## 五、风险与合规

1. **数据免责声明**: 已包含"数据仅供参考，不构成投资建议"，合规
2. **走势图误导风险**: 模拟数据未标注"模拟"，存在合规隐患
3. **数据源授权**: 非凸、AKShare、盈米数据的使用需确认各平台授权范围
4. **无用户数据收集**: 目前无登录/追踪，隐私风险低
