🧠 第3章：逻辑缺陷分析

缺陷1：无单一权威数据源（Single Source of Truth）

现状：15+ 个 JSON 文件存储重叠数据，不知道以哪个为准。

后果：

  • 修改 etf_standard_data.json → etf_merged_all_data.json 不同步
  • etf_prices.json 过期 → 前端可能显示旧价格
  • etf_risk_metrics.json 空 → 风险指标丢失

正确做法：只保留 1个 权威数据文件（etf_standard_data.json），其他都是临时缓存或备份。

缺陷2：pipeline step_build() 使用"填表格"模式

代码逻辑：

# pipeline.py step_build() 当前逻辑（错误） def step_build(): standard_data = [] # ← 创建空列表 for code in etf_codes: etf = {"code": code} # ← 创建空字典 etf["year_3_return"] = get_from_source(code) # ← 从数据源获取 # ... 其他字段 standard_data.append(etf) json.dump(standard_data, "etf_standard_data.json") # ← 覆盖！

问题：如果数据源缺少 year_3_return，etf["year_3_return"] 就是 0/null，覆盖后之前积累的数据就丢了！

正确做法：

# pipeline.py step_build() 应该的逻辑（正确） def step_build(): # 1. 加载已有数据 standard_data = json.load(open("etf_standard_data.json")) for etf in standard_data: # 2. 从数据源获取新数据 new_data = get_from_source(etf["code"]) # 3. 增量合并：只填充缺失/空字段 for key, value in new_data.items(): if key not in etf or etf[key] is None or etf[key] == 0: etf[key] = value # ← 只填充缺失的 # 4. 保存合并后的数据 json.dump(standard_data, "etf_standard_data.json")

缺陷3：数据更新策略混乱

现状：

  • etf_prices.json：过期11天，未更新
  • etf_prev_close.json：过期10天，未更新
  • etf_risk_metrics.json：空文件，未使用
  • etf_standard_data.json：每天被 pipeline build 覆盖（丢失数据）

正确做法：

  • ✅ etf_standard_data.json：权威源，增量合并，每日更新
  • ❌ 删除 etf_prices.json（价格应在 etf_standard_data.json 里）
  • ❌ 删除 etf_prev_close.json（昨收应在 etf_standard_data.json 里）
  • ❌ 删除 etf_risk_metrics.json（风险指标应在 etf_standard_data.json 里）

缺陷4：备份策略混乱

现状：根目录有 5+ 个备份文件，命名不规范（etf_standard_data_backup_20260521_075800.json）

正确做法：

  • ✅ 备份文件统一放在 data/backup/ 目录
  • ✅ 命名规范：etf_standard_data_YYYYMMDD_HHMMSS.json
  • ✅ 只保留最近 7 天日备 + 4 周周备
