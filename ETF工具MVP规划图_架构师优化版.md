```
┌─────────────────────────────────────────────────────┐

│  etf/templates/, stock/templates/, fund/...       │
└──────────────────┬──────────────────────────────────┘
                   │ 复用模板结构
┌──────────────────▼──────────────────────────────────┐
│  第3层：查询接口层（通用）                            │
│  api/query.py, api/compare.py, api/screen.py      │

                   │ 通用查询/对比/筛选
┌──────────────────▼──────────────────────────────────┐
│  第2层：数据模型层（每工具不同）                      │
│  etf/models.py, stock/models.py, base/models.py    │
└──────────────────┬──────────────────────────────────┘
                   │ 继承通用模型
┌──────────────────▼──────────────────────────────────┐
│  第1层：数据采集层（通用）                            │
│  financial_data_fetcher/ (base, multi_source, cache)│
└─────────────────────────────────────────────────────┘
```

### 1.2 分布式缓存架构（三地存储）

```
数据更新（单一入口）
        ↓
同步写入三个地点（原子性保证）
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 本地缓存  │    │  GitHub  │    │PythonAnywhere│
│(最快访问) │    │(版本控制)│    │  (线上服务)  │
└──────────┘    └──────────┘    └──────────┘
        ↓
智能调用策略（环境检测 → 最佳数据源 → 降级）
```

### 1.3 ETF重构为模板（Phase 1）

```
Week 1-2: 重构ETF工具（作为模板）

  → 创建 api/query.py（通用查询接口）
  → 测试确保ETF功能正常

Week 3-6: 扩展到其他金融工具
  → 股票工具（复用ETF架构）
  → 基金工具
  → 期货工具
  → 外汇工具
```
