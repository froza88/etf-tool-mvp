✅ 第3章：极简架构设计（像高铁票价）
🚄 极简架构：1 个权威文件 + 规范目录结构

核心原则：

  • ✅ 1 个权威文件：etf_standard_data.json（所有数据都在这里）
  • ✅ 0 个冗余文件：删除所有其他 JSON 文件（etf_merged_all_data.json 等）
  • ✅ 规范目录：data/ 目录存放所有数据相关文件
  • ✅ 增量合并：step_build() 改为"搭积木"模式
# 极简架构的目录结构 etf-tool-mvp/ ├── etf_standard_data.json # [权威源] 1.7MB，包含所有ETF的所有数据 ├── data/ │ ├── backup/ # 备份文件（7天日备 + 4周周备） │ │ ├── etf_standard_data_20260528.json │ │ ├── etf_standard_data_20260527.json │ │ └── ... │ ├── history/ # 历史K线数据（每ETF一个文件） │ │ ├── 510300.json │ │ ├── 510500.json │ │ └── ... │ ├── cache/ # 临时缓存（可删除） │ │ └── ... │ └── data_version.json # 数据版本信息 └── # 其他文件... # 删除的文件（冗余）： # - etf_merged_all_data.json (20MB) ❌ 冗余 # - etf_data_generated.json (1.1MB) ❌ 冗余 # - etf_calculated_metrics.json (499KB) ❌ 冗余 # - etf_yingmi_metrics.json ❌ 冗余 # - etf_risk_metrics.json (空) ❌ 废弃 # - etf_prices.json (过期) ❌ 冗余 # - etf_prev_close.json (过期) ❌ 冗余

⚠️ 风险提示

此操作会删除多个文件，修改核心逻辑。必须：

  1. ✅ 先完整备份当前所有数据
  2. ✅ 在测试环境验证后再部署到生产环境
  3. ✅ 保留回滚方案（可以随时恢复到备份）
步骤 操作 文件/命令 预计时间
1. 备份 备份当前所有数据到 data/backup/20260528_full/ cp -r *.json data/backup/20260528_full/ 5 分钟
2. 恢复 从最新备份恢复 etf_standard_data.json cp data/backup/etf_standard_data_20260521_131816.json etf_standard_data.json 2 分钟
3. 修复 修改 pipeline.py 的 step_build() 为增量合并 编辑 pipeline.py 30 分钟
4. 测试 运行 pipeline build，检查数据是否丢失 python3 pipeline.py build 10 分钟
5. 删除冗余 删除 etf_merged_all_data.json 等文件 rm etf_merged_all_data.json etf_data_generated.json ... 5 分钟
